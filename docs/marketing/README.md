# Marketing Docs

This folder is the website and product-messaging handoff package for Nomad Dashboard.

It is written for humans and AI builders that need to create a landing page, download page, product overview, or launch materials without re-exploring the repository.

## Documents

- [marketing.md](marketing.md): positioning, homepage copy direction, audiences, benefits, use cases, CTAs, and trust messaging
- [features.md](features.md): factual product inventory organized by feature area, including limitations and setup dependencies
- [brand-assets.md](brand-assets.md): brand system, asset links, visual direction, and website-builder guidance
- [../../Branding/README.md](../../Branding/README.md): source of truth for brand asset generation and exports

## Working Rules

- Keep Nomad Dashboard positioned as a native macOS app.
- Keep the product menu-bar-first, compact, and travel-oriented.
- State clearly that the app is free to download and use.
- State clearly that the app is provided as-is.
- Do not imply App Store distribution.
- Treat direct download and GitHub as first-class calls to action.
- Keep feature claims grounded in shipped capabilities.
- If mentioning future work, label it clearly as near-term roadmap, not current functionality.

## Core Product Facts

- Product: Nomad Dashboard
- Platform: macOS 14+
- Distribution: direct download outside the Mac App Store
- License: Apache-2.0

## Intended Use

Use these docs as a stable input contract:

- `marketing.md` for product story and homepage messaging
- `features.md` for factual capability mapping
- `brand-assets.md` for visual and asset guidance

If there is a conflict between aspirational website copy and the shipped app, prefer the shipped app.
